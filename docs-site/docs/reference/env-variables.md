# Environment variables

The portal reads its configuration from `.env`. The bundled `.env.example` enumerates every supported key. The install wizard (`scripts/install.sh`) populates the required keys with strong defaults; the rest you set as needed.

:::note Audience
Operators tuning a deployment. Familiarity with `.env` files and Docker Compose's variable substitution.
:::

## Reading order

1. `.env` in the repo root is loaded by `docker-compose` automatically.
2. Backend code calls `os.getenv()` **at runtime**, never at module import time. This is `CLAUDE.md` rule #11. Restarting the container is enough to pick up a changed value — no rebuild needed.
3. Compose substitutes `${VAR}` references in `docker-compose.yml` from `.env` at `docker-compose up` time.

## Required keys

These four must be present and non-empty. The wizard sets them.

| Key | Set by | Notes |
|---|---|---|
| `SECRET_KEY` | wizard (`openssl rand -hex 32`) | JWT signing key. Rotating invalidates every refresh token. |
| `DATABASE_URL` | wizard | `postgresql+asyncpg://user:pass@postgres:5432/trustedoss`. Must use the `postgres` host (compose service name). |
| `CORS_ALLOWED_ORIGINS` | wizard | Comma-separated. Production must enumerate origins explicitly — no wildcard fallback. |
| `DOMAIN` | wizard | Hostname used by Traefik's host-rule. Stripped of scheme and path. |

## Application

| Key | Default | Description |
|---|---|---|
| `APP_ENV` | `dev` | `dev` or `prod`. Enables stricter defaults in `prod` (CSP, secure cookies). |
| `LOG_LEVEL` | `INFO` | `DEBUG`, `INFO`, `WARNING`, `ERROR`. |
| `IMAGE_TAG` | `2.0.0` | Pinned tag for `trustedoss/backend`, `trustedoss/backend-worker`, `trustedoss/frontend`. |

## Database

| Key | Default | Description |
|---|---|---|
| `DATABASE_URL` | — | See above. |
| `POSTGRES_USER` | `trustedoss` | Used by the postgres container's init. Must match `DATABASE_URL`. |
| `POSTGRES_PASSWORD` | — | Generated by the wizard. |
| `POSTGRES_DB` | `trustedoss` | Database name. |

The portal uses async SQLAlchemy + `asyncpg`. Connection pool defaults are tuned for the FastAPI worker count (4 uvicorn workers × 5 connections each = 20).

## Redis & Celery

| Key | Default | Description |
|---|---|---|
| `REDIS_URL` | `redis://redis:6379/0` | Broker + result backend. |
| `CELERY_CONCURRENCY` | `2` | Worker process count. Each slot needs ~2 GB RAM at peak. |

## Authentication

| Key | Default | Description |
|---|---|---|
| `SECRET_KEY` | — | See [Required keys](#required-keys). |
| `ACCESS_TOKEN_EXPIRE_MINUTES` | `30` | JWT access token lifetime. |
| `REFRESH_TOKEN_EXPIRE_DAYS` | `7` | Refresh token lifetime. Rotation + reuse detection enabled. |

## Dependency-Track

| Key | Default | Description |
|---|---|---|
| `DT_URL` | `http://dtrack-api:8080` | DT API base URL. Use the compose service name when DT is bundled, or the public URL when external. |
| `DT_API_KEY` | — | Issued by DT's Automation team. **Required** for vulnerability data. Without it, the breaker stays OPEN and scans use the cache. |
| `DT_ORPHAN_AUTODELETE` | `false` | When `true`, orphan-cleanup deletes DT projects automatically; default requires operator confirmation. |

See [DT connector](../admin-guide/dt-connector.md) for the bootstrap flow.

## Workspace & ORT

| Key | Default | Description |
|---|---|---|
| `WORKSPACE_HOST_PATH` | `/opt/trustedoss/workspace` | Host directory mounted into the worker as `/workspace`. Holds repo clones + ORT analyzer outputs. |
| `ORT_RULES_PATH` | `/opt/trustedoss/ort/rules.kts` | Path inside the worker. Mount your customized rules here. |

## Notifications

| Key | Default | Description |
|---|---|---|
| `SMTP_HOST` | (empty) | SMTP server. Without it, email notifications are skipped. |
| `SMTP_PORT` | `587` | SMTP port. STARTTLS expected on 587. |
| `SMTP_USER` | (empty) | SMTP username. |
| `SMTP_PASSWORD` | (empty) | SMTP password. |
| `SMTP_FROM` | `noreply@<DOMAIN>` | Envelope-from address. |
| `SLACK_WEBHOOK_URL` | (empty) | Org-wide Slack webhook for `super_admin` notifications. Per-team webhooks are configured in the UI. |
| `TEAMS_WEBHOOK_URL` | (empty) | Org-wide MS Teams webhook. |

| Key | Default | Description |
|---|---|---|
| `OTEL_EXPORTER_OTLP_ENDPOINT` | (empty) | When set, the backend exports OTLP traces. |
| `OTEL_SAMPLER` | `parentbased_always_off` | Set to `parentbased_always_on` to enable trace sampling. |

## Validation

The backend validates the configuration on startup:

- Refuses to start if `SECRET_KEY` is shorter than 32 chars.
- Refuses to start if `DATABASE_URL` does not use `postgresql+asyncpg`.
- Refuses to start in `APP_ENV=prod` if `CORS_ALLOWED_ORIGINS` is empty or contains `*`.
- Logs a warning (not fatal) if `DT_API_KEY` is empty — the breaker stays OPEN.

Failures emit a structured log line with `event=config.invalid` and the offending key.

## Verify it worked

After editing `.env`:

```bash
docker-compose -f docker-compose.yml restart backend worker beat
docker-compose -f docker-compose.yml logs --tail=50 backend | grep -i config
```

The startup log should print `event=config.loaded` with the keys (values are not logged for secrets).
