# v2.0.0 — 정식 출시(GA)

**릴리스 날짜:** 2026-05-09

TrustedOSS Portal의 첫 정식 출시(General Availability)입니다. `2.0.0-rc.1`을 승격하고 rc 이후 정리 웨이브(PR #28 ~ #31)를 흡수했습니다.

:::tip rc.1에서 업그레이드하시나요?
마이그레이션은 자동 적용되며 새로 추가된 필수 `.env` 변수는 없습니다. pull 후 backend 컨테이너에서 `alembic upgrade head` — 자세한 내용은 [rc.1에서 업그레이드](#rc1에서-업그레이드)를 보세요.
:::

## 추가됨

### 인증 UX (PR #28)

- **`/forgot-password`**가 백엔드와 연결되었습니다 — 이메일을 제출하면 24시간·1회용 리셋 링크를 받습니다.
- **`/reset-password?token=…`**는 이메일의 토큰을 받아 새 비밀번호를 설정하는 새 페이지입니다.
- **`i18next-parser` 드리프트 게이트**가 CI에 도입되었습니다 — 매칭되는 KO 키 없이 EN 키만 추가하면 PR이 실패합니다.
- **`/login`의 OAuth** — GitHub·Google 버튼, 최초 로그인 시 개인 팀 자동 프로비저닝, `(provider, provider_user_id)` 기반 신원 매칭, 7개 오류 코드 i18n 매핑, `redirect_after` 통과.

> 함께 보기: [인증과 프로필](../user-guide/auth-and-profile.md).

### `/integrations` 페이지 (PR #28)

- 셀프 서비스 API Key 목록·생성·폐기 흐름.
- 1회 평문 노출 모달과 클립보드 복사 버튼, 전체 Key가 단 한 번만 표시된다는 강한 경고.
- Webhook URL 정보 — `https://<host>/v1/webhooks/github`(HMAC `X-Hub-Signature-256`), `https://<host>/v1/webhooks/gitlab`(token 헤더).

> 함께 보기: [통합](../user-guide/integrations.md).

### 백업 자동화 (PR #29)

- **Celery Beat가 매일 00:00 UTC에 백업** 실행, `auto-*` 백업은 **7일 자동 보존**.
- **`/admin/backup` UI** — 수동 트리거, 스트리밍 다운로드, `restore` 타이핑 게이트가 있는 Upload+Restore(`X-Confirm-Restore` 헤더 동반), 삭제(auto-* 보호).
- 탭 포커스 시 `useScanWebSocket`이 즉시 재연결 — 다음 재연결 틱을 기다리지 않고 스캔 진행 업데이트가 재개됩니다.

> 함께 보기: [백업·복원](../admin-guide/backup-and-restore.md).

### Locust 부하 하네스 (PR #30)

- `tests/load/`와 `docker-compose.load.yml`로 staging 한정 **p95 < 1 s SLO** 검증.
- CI에 연결되지 않음 — staging 환경에서 필요 시 실행.

### SCA self-scan (PR #30)

- `sca-self.yml` 야간 워크플로가 `cdxgen`으로 CycloneDX SBOM을 생성하고 Trivy로 스캔, **CRITICAL 취약점 발견 시 GitHub 이슈를 자동으로 열고 닫습니다**.
- 포털이 자기 도그푸드를 먹기 시작했습니다.

### API Keys + Webhooks 테스트 커버리지 (PR #31)

- **테스트 125개**(unit 52 + integration 30 + 23 + 20)로 `services/api_key_service.py`의 line coverage를 **0 %에서 88.24 %**까지 끌어올림.
- 모든 webhook URL·서명·SPDX 표현식 파서에 적대적 입력 parametrize 매트릭스 적용.

## 변경됨

### SAST CI는 이제 HARD FAIL (PR #30)

- **`bandit`**은 **High+**에서 차단.
- **`semgrep`**은 **ERROR**에서 차단.
- **Trivy 이미지 스캔**은 **CRITICAL**에서 차단.
- HIGH Trivy 발견은 Phase 8 worker 이미지 갱신 전까지 권고로 유지.

## 수정됨

- `tasks.backup`의 `scripts/` 지연 해석 — 모듈이 `/app/tasks/` 컨테이너 내부에서 깨끗하게 import됩니다. 이전 `parents[3]` 조회는 컨테이너 경로 깊이를 초과했습니다(PR #29).
- `.semgrepignore`와 인라인 정당화로 **21개**의 semgrep ERROR 처리 — 어떤 것도 실제 위험을 의미하지 않습니다. `.semgrepignore`와 라인별 `# nosemgrep` 주석에 문서화(PR #30).
- SAST 워크플로에 `setuptools<78`을 핀 — `semgrep`이 transitive하게 끌어오는 `opentelemetry` 패키지에서 `pkg_resources`가 import 가능하게 유지(PR #30).

## rc.1에서 업그레이드 {#rc1에서-업그레이드}

업그레이드는 단순합니다 — 2.0.0에는 **새 필수 환경 변수가 없으며** 마이그레이션은 forward-only로 적용됩니다.

```bash
cd /opt/trustedoss-portal
git pull origin main

# 새 이미지 가져오기
docker-compose -f docker-compose.yml pull

# 마이그레이션 적용 (이미 head라면 no-op)
docker-compose -f docker-compose.yml exec backend alembic upgrade head

# 새 이미지로 재시작
docker-compose -f docker-compose.yml up -d
```

소규모 설치라면 마이그레이션 윈도가 60초 미만입니다. 기존 JWT 세션은 유효하게 유지되며 사용자가 다시 로그인할 필요는 없습니다.

새 매일 백업을 사용하려는 경우 별도 작업이 필요 없습니다 — `BACKUP_DAILY_ENABLED=true`로 기본 활성화되어 있습니다.
